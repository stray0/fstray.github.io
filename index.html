HashMap是线程安全的吗？

HashMap不是线程安全的，可以采用HashTable，但是效率较低，若想提升并发时的效率，可以采用ConcurrentHashMap。

扩容机制

在初始化HashMap的时候我们可以指定HashMap的容器大小以及负载因子（当Map内个数大于容器大小*负载因子时，会进行扩容），若不指定，则默认的容器大小为16，负载因子为0.75。
上面的指定可能用的不恰当，因为并不是我们输入多少，HashMap就生成相应的容器给我们，而是有一定的机制，下面是相应的源代码 

static final int tableSizeFor(int cap) {
    int n = cap - 1;
    n |= n >>> 1;
    n |= n >>> 2;
    n |= n >>> 4;
    n |= n >>> 8;
    n |= n >>> 16;
    return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

输入是我们指定的n，输出是大于等于n的2的x次方（小于最大容器值），若输入7，则返回8；输入16，则返回16。
